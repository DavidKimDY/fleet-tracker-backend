# Fleet Tracker Backend 명세

WebSocket 기반 GPS 추적 서버의 동작을 정의한다. 모든 데이터는 RAM에 저장하며, 식별자 단위로 GPS·속도·가속도 정보를 관리한다.

---

## 데이터 구조

식별자별로 시간을 키로 하는 중첩 JSON 형태로 저장한다.

```json
{
  "식별자": {
    "시간": {
      "GPS": "...",
      "속도": "...",
      "가속도": "..."
    }
  }
}
```

| 필드 | 설명 |
|------|------|
| `식별자` | 요청자(또는 추적 대상)를 구분하는 고유 값 |
| `시간` | GPS 수신 시각 |
| `GPS` | GPS 좌표 정보 |
| `속도` | 이전 GPS·시간 대비 계산된 속도 |
| `가속도` | 이전 속도·시간 대비 계산된 가속도 |

---

## 서버 행동

### 1. GPS 정보 수신 및 저장

**요청:** 요청자가 WebSocket으로 식별자와 GPS 정보를 전송한다.

**처리:**

1. 해당 식별자의 GPS 정보를 RAM에 저장한다.
2. 저장된 GPS 정보와 시간 정보를 이용해 **속도**를 계산하여 함께 저장한다.
3. 저장된 속도 정보와 시간 정보를 이용해 **가속도**를 계산하여 함께 저장한다.

**비고:** 최초 수신 시점에는 이전 데이터가 없으므로 속도·가속도 계산 방식은 구현 단계에서 정의한다.

---

### 2. GPS 정보 조회

**요청:** 요청자가 WebSocket으로 특정 식별자에 저장된 GPS 정보를 요청한다.

**응답:** 해당 식별자에 저장된 GPS 데이터를 JSON 형태로 반환한다.

---

### 3. 속도 정보 조회

**요청:** 요청자가 WebSocket으로 속도 데이터를 요청한다.

**응답:** 해당 식별자에 저장된 속도 데이터를 JSON 형태로 반환한다.

---

### 4. 식별자 저장 한도 (FIFO)

RAM에는 **최대 5개**의 식별자 데이터만 유지한다.

- 새로운 식별자가 **6번째**로 저장될 때, **가장 오래된(선입) 식별자**의 데이터를 삭제한다.
- 선입선출(FIFO) 방식으로 동작한다.

```
[식별자1] → [식별자2] → ... → [식별자5]
                                    ↓ 6번째 식별자 저장 시
                              [식별자1] 삭제
```

---

### 5. RAM 전체 조회 (HTTP)

**요청:** `GET /ram`

**응답:** 현재 RAM에 저장된 모든 식별자와 데이터를 JSON 형태로 반환한다.

```json
{
  "식별자": {
    "시간": {
      "GPS": "...",
      "속도": "...",
      "가속도": "..."
    }
  }
}
```

---

## 통신 방식

- **WebSocket** (`/ws`): GPS 수신, GPS·속도 조회
- **HTTP** (`GET /ram`): RAM 전체 조회

---

## 구현 시 확인 사항

아래 항목은 명세에 명시되지 않았으므로 개발 과정에서 결정한다.

- WebSocket 메시지 형식 (요청/응답 스키마)
- GPS 좌표 표현 형식 (위도·경도, 문자열/숫자 등)
- 속도·가속도 계산 공식 및 단위
- 최초 GPS 수신 시 속도·가속도 처리 방식
- 속도 조회 요청 시 식별자 지정 방식
- 시간 필드 형식 (Unix timestamp, ISO 8601 등)
